/speckit-plan Implement in Python 3.11+, one repo, one codebase, on Windows. Storage is local: SQLite and/or Parquet, append-only for raw market events — no Kafka, Redis, or microservices. Tests with pytest.
The provisional venue is Bybit testnet, reached only through a strict ExchangeClient interface, with all Bybit-specific code confined to a single adapter module (src/trading/execution/adapters/bybit_testnet.py) so that swapping to a Canada-legal venue (Kraken/Coinbase) later is a one-module change. No venue-specific types, payloads, enums, or error shapes may leak above the adapter into data, strategy, risk, or backtest.
Enforce the four-layer import boundaries in CI with import-linter, wired into the test command so the build FAILS if risk/ imports any ML/AI library, or if strategy/, risk/, data/, or backtest/ import execution adapters. Set these contracts up in the very first task, while the modules are still near-empty — this is the load-bearing guardrail, not a later refinement.
Repo layout: src/trading/{data,strategy,risk,execution,backtest,logkit,loop}; config in config/config.yaml; secrets only in a gitignored .env with .env.example holding variable NAMES not values; raw events in data/ and logs in logs/, both gitignored; tests in tests/ including test_risk.py and test_backtest_costs.py.
A TRADING_ENV flag defaults to testnet, and the execution client MUST refuse to construct a live/mainnet connection without an explicit, deliberate override — it must be impossible to place a real-money order by accident.
The risk engine is pure and deterministic: no I/O, no network, no randomness, no clock reads buried inside logic (pass time in as a parameter). It returns pass/clamp/veto, where a clamp may only reduce size toward zero — never increase size, never flip direction. Hard limits: max position size (configurable, default 1 BTC), max daily loss (configurable as a percentage of configured account equity, default 5%), and a kill switch that blocks new orders while still permitting cancellation logic to run.
Execution operational defaults: idempotent client order IDs, exponential backoff on HTTP 429 rate limits, graceful degradation on venue outage.
Every decision record includes at minimum: UTC timestamp, layer, event_type, reason_code, venue, symbol, side, size, intended vs executed price, fees, strategy_version, and feature_snapshot_hash. The ledger additionally captures CAD value for Canadian tax records. Reason codes are a controlled vocabulary in the shape LAYER_VERB_DETAIL (e.g. STRAT_NO_SIGNAL, RISK_PASS, RISK_CLAMP_MAX_POSITION, RISK_VETO_DAILY_LOSS, RISK_VETO_KILL_SWITCH, EXEC_ORDER_SUBMITTED, EXEC_ORDER_FILLED, EXEC_BACKOFF_RATE_LIMIT). No secret ever appears in a log line or decision record.
Costs: fees default to 0.1% taker per side (configurable); the backtest applies fees + bid/ask spread + a slippage model to every simulated trade, with no cost-free code path in existence.
Expected contracts/: the Strategy interface (decide(market_state) -> DesiredPosition) and the ExchangeClient interface. Expected data-model.md entities: MarketState, DesiredPosition, ApprovedOrder, DecisionRecord, Trade/Fill.
