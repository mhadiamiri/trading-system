ROOT CAUSE + FIX-SCOPE FINDING (the count is LARGE -> §1 says it comes back for a ruling)
================================================================================
The race is NOT 30 independent test defects. It is ONE structural fact: get_live_market_data terminates
ONLY on `while time.time() < deadline` (a REAL wall clock, line 2434), and the tests drive it against that
real clock. Every test that asserts on events processed inside the loop therefore gambles against
scheduler load. Widening deadlines is refused (§2, tuning-to-green). The three ways to make it
deterministic:
  (a) INJECTABLE CLOCK for the deadline — line 2388 uses raw time.time(), not self._wall_clock; route it
      through an injectable clock and tests advance it deterministically. PRODUCTION CHANGE.
  (b) DETERMINISTIC CAPTURE-END SIGNAL in the harness — extend fake_ws_transport with a scripted
      "capture complete" sentinel the transport honors as a clean end after the scripted events.
      PRODUCTION CHANGE (the loop must honor the signal).
  (c) monkeypatch time.time in tests — REFUSED (§2: patch-based mitigation of a structural problem).
So the only non-refused fixes (a)/(b) TOUCH PRODUCTION CODE — exceeding this WO's "tests + docs only,
expected NO ship impact" framing (§6). Per §1, with a LARGE count and a fix that changes production, the
fix scope comes back for a RULING rather than expanding silently. Recommendation for the ruling: option
(a) — a single injectable-clock seam on the deadline — is the smallest, most surgical production change,
closes all 30 races by construction, and is order/load-independent by design, not by margin.
