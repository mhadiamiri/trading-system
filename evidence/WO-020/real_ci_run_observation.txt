ROOT CAUSE (confirmed in source):
  - src/trading/data/adapters/kraken_v2_book.py uses `AsyncIterator[MarketState]` as a return-type
    annotation at line 2300 (and line 2718) but NEVER IMPORTS it. Line 20 imports only
    `from typing import Optional, List, Dict`. No `from __future__ import annotations`.
  - Python 3.11 evaluates annotations EAGERLY at class-definition time -> the name `AsyncIterator`
    is looked up when the class body runs -> NameError at import -> collection fails (31 errors).
  - Python 3.14 defers annotation evaluation (PEP 649) -> the annotation is never evaluated at def
    time -> the missing import is MASKED -> the local 3.14 clean-venv run (WO-019) passed with 215.
  This is exactly why WO-019's local 3.14 reproduction could not surface it, and why the H2 leg
  (which needed 3.11) was the missing piece.

THE FIX (do NOT apply here — awaits the lead's ruling): import AsyncIterator (from collections.abc,
or typing), or add `from __future__ import annotations` to the module. A trivial change to ONE
production file — but it is production code, in scope for the version-ruling successor, not WO-020.
