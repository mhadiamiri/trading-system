── VALIDATED DESIGN (linchpin proven this session; see the WireDecimal probe) ────────────────
1. class WireDecimal(Decimal): __new__ stores the raw token when it is a str; `.wire` returns it.
   PROVEN: json.loads(parse_float=WireDecimal) -> qty.wire == "0.00000010" (transmitted text, NOT
   str(Decimal)="1.0E-7"); survives into the (price,qty) tuple and through sort() untouched; a plain
   Decimal has no `.wire`. Arithmetic would drop the subclass, but the BOOK never does arithmetic on
   a level — apply REPLACES — so the WireDecimal survives from parse to checksum.
2. Live parse (:2506): json.loads(message, parse_float=WireDecimal, parse_int=WireDecimal).
3. _parse_levels: str -> WireDecimal(str); WireDecimal -> as-is; else RAISE (declared code). §1.2 is
   a PRIVATE representation change (ladder still holds (Decimal-subclass, Decimal-subclass) tuples) —
   NO public signature change, so 0.1a is not triggered; if some public signature must change, STOP.
4. _current_ladder_strings (§1.3): return (p.wire, q.wire) — NO str()/format(). If a level lacks
   .wire -> raise CHECKSUM_WIRE_STRING_MISSING (§1.4 NO FALLBACK, load-bearing). The INTERIM
   format('f') path is REMOVED here (§3), not left dead.
5. compute_checksum: UNCHANGED (already consumes strings). The 'E'-sentinel STAYS (§3) — it now
   guards the invariant "no synthesized notation reaches the CRC"; confirm it still bites.
6. New reason code CHECKSUM_WIRE_STRING_MISSING (declared same commit).
