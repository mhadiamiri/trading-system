WO-012c §1 — PRESERVATION DENOMINATOR (derived by reading for SHAPE, not grepped)
=================================================================================
METHOD: for each of WO-012's 12 refusal surfaces, ask "what must STILL WORK when
this guard fires?" — semantic reading, not phrase matching. Every dual below was
found by SEMANTIC READING; the WO-012b phrase-sweep ("must still permit / must not
block / ...") found only S12/S13. Method is annotated per row so the denominator's
own reliability is auditable.

BEYOND THE 12 — free-standing preservation guarantees (not duals of any refusal)
--------------------------------------------------------------------------------
- Observability / "zero silent decisions" (Principle VIII): a positive REQUIREMENT to
  always log, not a GUARD that fires and could over-block. No over-block failure mode;
  not a preservation guarantee in this sense. Enforced by the logger + review.
- FR-023/024/025 "MUST remain unchanged": WO-012b classified these as INTERFACE-
  STABILITY (development) constraints, not runtime preservations. CLASSIFICATION
  CONFIRMED — they describe what the code may not become, not what must still run when
  a guard fires. No free-standing runtime preservation guarantee found.
