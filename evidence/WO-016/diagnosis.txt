── §1.2 REPEATED-PRICE SEMANTICS, BY CITATION (0.1e) — see repeated_price_semantics.txt ───────
Kraken v2 book checksum guide (docs.kraken.com/api/docs/guides/spot-ws-book-v2), verbatim:
  "Process all price level updates in a message before calculating the checksum."
  "Parse price and qty fields using a decimal or string decoder to preserve full precision
   through deserialisation."
  "Remove . from price" / "Remove leading zeros" / "Remove . from qty" / "Remove leading zeros".
DOCUMENTED SILENCE on the explicit term "last-wins" for a price repeated within one update; but
"process all updates before the checksum" means SEQUENTIAL application (later entry for the same
price supersedes the earlier). BEHAVIORAL EVIDENCE (sovereign for what happened): our last-wins
apply produced a book whose fixed-point checksum matches Kraken's EXPECTED for all 55 repeated-
price captures (part of the 200/200 below) ⇒ our repeated-price apply is CORRECT. Inference
labelled: repeated prices are a correlated SYMPTOM (a level reduced to a tiny residual qty), not
the cause.

── ROOT CAUSE (specific line) ────────────────────────────────────────────────────────────────
src/trading/data/adapters/kraken_v2_book.py, _current_ladder_strings (≈ line 1216-1217):
    bid_levels = [(str(p), str(q)) for p, q in self._local_book.bids[:10]]
    ask_levels = [(str(p), str(q)) for p, q in self._local_book.asks[:10]]
`str()` on a Decimal such as Decimal('1.0E-7') returns the SCIENTIFIC string '1.0E-7'. compute_
checksum then does str.replace('.', '').lstrip('0') → '10E-7' instead of Kraken's '10'. A single
malformed level corrupts the whole CRC. This violates Kraken's cited rule to "preserve full
precision" with a decimal/string decoder — str() does not yield a plain decimal for small values.
(The ground-truth vector passed because validate_kraken_ground_truth feeds decimal STRINGS
directly, never exercising the Decimal→str scientific path — a real gap the live feed first hit.)

── PROPOSED FIX (STATED — NOT IMPLEMENTED; §2 is gated behind approval, rule 0.9 seam) ────────
Format Decimal price/qty in FIXED-POINT (precision-preserving) notation at the single Decimal→
string site, e.g. in _current_ladder_strings use `format(q, 'f')` (and `format(p, 'f')`) instead
of `str(q)`. `format(Decimal, 'f')` never emits scientific notation and preserves the value.
Verified: yields Kraken's expected checksum 200/200.
