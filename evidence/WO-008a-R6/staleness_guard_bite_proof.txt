REASON CODES USED
=================

Consistent with existing rejection reason code convention:
- EXEC_NO_MARKET_STATE (no state registered)
- EXEC_STALE_MARKET_STATE (state too old)

Pattern: EXEC_* for execution layer rejections (matching EXEC_ORDER_FILLED, EXEC_BLOCKED_KILL_SWITCH)

---

CONFIGURATION
=============

Threshold: 18 seconds (DEFAULT_STALENESS_THRESHOLD_SECONDS)

Reasoning:
- Historical rate: ~10 MarketStates/min → 1 state every 6 seconds
- Threshold: 18 seconds = 3x historical interval
- Allows normal variance (brief gaps common in WebSocket feeds)
- Detects genuine stalls within seconds
- Not a round number: derived from 3 × (60 / 10) = 18
- Tight enough to detect stalls quickly, loose enough to avoid constant rejection

Configurable via: PaperExecutionClient(staleness_threshold_seconds=N)
