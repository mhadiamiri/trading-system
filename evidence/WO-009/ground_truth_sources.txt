################ ⚠ ANSWER TO THE §2.3 QUESTION — STATED PLAINLY ################
IS GROUND TRUTH AVAILABLE FOR INCREMENTALS? **NO.**

Kraken's documentation publishes a checksum for the SNAPSHOT case ONLY.
There is no documented UPDATE message carrying a checksum value.

Therefore INDEPENDENT verification of the INCREMENTAL path is NOT possible
from documentation. Per §2.3 this is stated plainly rather than substituting
another self-generated value and labelling it ground truth.

CONSEQUENCE: incremental checksum verification becomes the job of FIRST LIVE
CONTACT in WO-008b-A. Until a real Kraken update frame is captured with its
own checksum, the incremental path is unverified against reality — however
many self-consistent fixtures exist.

This matters more than usual here: WO-008b-DIAG established that the current
implementation validates the checksum against the PRE-update book while
Kraken defines it over the POST-update book. A self-generated fixture cannot
catch that class of error, because our code would make the same mistake on
both sides of the comparison.
